# Claude Code 项目入口

本仓库项目事实、能力边界、路线图、任务清单与验证口径统一维护在根目录 `README.md`。进入本仓库后的第一步必须阅读 `README.md`，不要再把 `docs/mvp/`、`docs/ui/`、`docs/superpowers/` 中的历史文件当作当前状态来源。

## 固定沟通约定

- 默认使用中文回复用户。
- 代码注释、提交信息和文档默认中文，除非现有代码上下文要求英文标识。
- 涉及“是否已实现 / 是否可对外承诺”的表述，必须以 `README.md` 的能力边界为准。

## 当前不可突破的对外口径

当前只可承诺：**前台 / 应用存活期间的采样能力 + 本地补发思路 + 手动 SOS**。

不得承诺：Android 长时后台保活、被系统杀死后持续追踪、长期前台服务方案、自动报警、围栏、异常检测、正式应急平台、完整账号体系、实时地图监护或地图化历史回放已经完成。

## 常用入口

- 统一项目文档：`README.md`
- 前端目录：`frontend/`
- 后端目录：`backend/`
- API 专项契约：`docs/api/README.md`
- 原始愿景来源：`神秘组织内部资料.md`

## 不要碰的东西

- `*.jks` / `*.keystore` / `release-signing.properties`：签名资产，不要读取、复制或写入仓库。
- `.env` / `.env.*`：未入库；新增前先和用户对齐。
- `backend/data/safety.db`：本地数据库文件，任何删除或重建都要明确告知用户。

## 文档维护规则

- 当前项目状态、能力边界、路线图、任务清单、审查结论只更新 `README.md`。
- 专项文档只能承载专项契约或历史说明，不得重新形成平行事实来源。
- 历史 agent 文档已归档；如需恢复其中内容，先并入 `README.md` 对应章节，再删除旧入口表述。
