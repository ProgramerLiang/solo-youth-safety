>
> P0-3 统一边界摘要：当前可承诺能力仅为 **前台 / 应用存活期间的周期采样 + 本地补发 + 手动 SOS**；当前不承诺 Android 长时后台保活、被杀后持续追踪、前台服务长期运行方案、自动报警 / 围栏 / 异常检测 / 正式应急平台。

- 远端模式已完成 **MVP 级最小授权边界**：请求头身份声明、最小归属校验、跨用户资源拒绝与 CORS 白名单收口已落地。
- P0-2 **SOS 失败恢复与用户可见状态机** 已完成：前端已落地定位失败、远端失败、短信失败、拨号失败、部分成功等状态机分支；SOS 页可展示结果摘要、子步骤状态、失败原因，并提供重试 / 重新获取位置 / 仅拨号 / 仅短信等补救动作；本地 / 远端 `notifications` 也已统一到细粒度状态摘要层。
- 简洁事实摘要：本轮已完成的是“手动触发 SOS 后的状态透明化 + 失败恢复闭环”，不是“自动告警编排平台”；原生短信 / 拨号状态口径固定为“已尝试发送 / 已尝试拉起”，不表示送达确认或接通确认。
- 相关 README、前端 README、后端 README、API 文档、任务清单与路线图已同步更新，不再继续使用“P0-2 待开始”或“P0-2 待收口”的旧口径。
- 统一限制说明：当前仍未引入正式账号体系、token、session、多租户组织模型、家庭共享授权流；Android 侧当前也仅支持前台 / 应用存活期间的周期采样 + 本地补发 + 手动 SOS，未完成 Android 长时后台保活、被系统杀死后仍持续追踪、前台服务长期运行方案、自动报警或正式应急平台编排，因此不能把本项目表述为“正式安全后端”“完整授权系统”或“完整守护平台”。
- 下一阶段正式进入 **P0-3：Android 后台限制 / 前台服务边界澄清**；安全方向的后续工作继续保留在 HTTPS、部署模板、设备管理、审计与共享授权流。
