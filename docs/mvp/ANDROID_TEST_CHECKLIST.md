# Android 实机测试清单（MVP）

## A. 首次启动与权限
- [ ] 首次打开 App 自动弹出定位权限请求
- [ ] 点击“允许”后页面显示“已获取当前位置”
- [ ] 点击“拒绝”后页面显示“定位权限未授予”且 App 不崩溃
- [ ] 进入“总览”页后可看到当前位置、新鲜度、定位精度、最近刷新时间
- [ ] 点击“刷新当前位置”后状态文案、定位精度与最近刷新时间会更新
- [ ] 手动刷新时会进行 2~3 次采样，并自动采用精度最佳结果
- [ ] 默认显示“总览 / 守护 / 通知配置 / 联系人 / SOS / 历史”主流程页面跳转入口
- [ ] 顶部页面跳转条可直接切换不同页面
- [ ] 点击左上角菜单按钮可呼出侧边栏，并可切换不同页面
- [ ] 在页面空白处从左向右滑动可呼出侧边栏
- [ ] 侧边栏打开后可左滑收起，且拖拽过程跟手、无明显误触
- [ ] 切换页面后会自动回到顶部，不会停留在上一页滚动位置
- [ ] 当前页面主体宽度约为 80%~90% 屏宽，左右保留可操作空白区
- [ ] 连续点击版本号 5 次后可开启开发者模式并显示“工具”页
- [ ] 窄屏下导航不遮挡主要操作，页面可正常滚动

- [ ] 可编辑联系人并保存新号码
- [ ] 可删除联系人

## C. 主题与外观

## D. 轨迹守护与补发
- [ ] 在“守护”页可看到“轨迹守护”区块、周期、待补发数量、最近采样/同步时间
- [ ] 点击“开启周期轨迹”后，App 会按设定周期自动采样并写入轨迹
- [ ] 可切换采样周期（30 / 60 / 180 / 300 秒）并立即持久化
- [ ] 点击“立即采样并补发”后，待补发队列数量会正确变化
- [ ] 在远端后端不可用时，轨迹写入失败会进入本地待补发队列
- [ ] 恢复网络或重新回到前台后，待补发队列可自动继续补发

## E. SOS 倒计时流程
- [ ] 点击“触发 SOS”进入 5 秒倒计时

- [ ] 定位精度较弱时，SOS 页会显示精度提醒 banner
- [ ] 倒计时结束后会先尝试刷新旧位置 / 缺失位置，再触发 SOS
- [ ] SOS 前自动刷新时会进行 2~3 次采样，并自动采用精度最佳结果

## H. Android 10+/13+/14+ 边界专项验证（最小模板）

### H1. 适用范围与当前口径
- [ ] 本专项至少覆盖 Android 10、Android 13、Android 14 各 1 台真机；若同版本多机型结果不一致，需分别记录
- [ ] 记录前先确认当前 APK/版本号、测试时间、测试人员、网络环境（Wi‑Fi/4G/5G）
- [ ] 记录前先确认 App 当前实现口径：**仅验证应用存活期间的定位采样、失败入队与回前台后的补发，不视为长期后台保活承诺**
- [ ] 记录前先确认当前 Android Manifest 未声明 `ACCESS_BACKGROUND_LOCATION`、`FOREGROUND_SERVICE`、`FOREGROUND_SERVICE_LOCATION`、`RECEIVE_BOOT_COMPLETED`、`WAKE_LOCK`、`REQUEST_IGNORE_BATTERY_OPTIMIZATIONS`
- [ ] 若现场行为超出上述实现基础，必须附日志/截图并单独标记“需复核”，不得直接写成已承诺能力

### H2. 权限与系统开关分档核验
- [ ] Android 10：记录定位权限是否仅“使用期间允许”/是否出现“始终允许”相关入口，以及实际选择项
- [ ] Android 13：记录定位权限实际授权结果（精确/大致、使用期间允许/拒绝）
- [ ] Android 14：记录定位权限实际授权结果（精确/大致、使用期间允许/拒绝）
- [ ] Android 13+：记录通知权限是否弹窗申请、是否授予、拒绝后是否影响普通页面使用
- [ ] 记录系统“电池优化”状态（默认 / 未受限 / 受限 / 厂商省电策略名）
- [ ] 记录系统是否存在“自启动 / 后台弹出界面 / 锁屏清理 / 省电策略白名单”等厂商定制开关，以及本次是否修改
- [ ] 明确填写：当前版本**未承诺**后台定位常驻、被系统杀死后持续追踪、开机自启恢复、长期前台服务持续运行

### H3. 场景检查项
- [ ] 切到后台 5 分钟内：记录是否继续采样、是否新增待补发、是否出现异常提示
- [ ] 熄屏后静置 10~15 分钟：记录是否停止采样、是否出现延迟、恢复亮屏后是否补发
- [ ] Doze/待机场景（可用 `adb shell dumpsys deviceidle force-idle` 或人工长时间静置替代）：记录采样、写入、补发是否受影响
- [ ] 从最近任务划掉 App：记录是否立即停止采样、是否保留本地待补发、重新进入后是否恢复补发
- [ ] 系统设置页或 `adb shell am force-stop` 强杀 App：记录是否停止采样、重新打开后是否仅执行回前台后的补发
- [ ] 网络断开后切后台再回前台：记录失败入队数量、恢复网络后是否自动补发
- [ ] Android 13+/14+ 通知权限拒绝时：记录是否出现前台服务通知、是否出现常驻通知、是否有系统限制提示
- [ ] 整个过程中，明确记录**是否出现前台服务通知**；若未出现，应填写“未出现，符合当前未实现前台服务方案的口径”
- [ ] 恢复到前台后：记录轨迹队列是否自动补发、补发完成耗时、是否需要手动点击“立即采样并补发”
