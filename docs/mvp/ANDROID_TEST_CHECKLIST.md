# Android 实机测试清单（MVP）

## A. 首次启动与权限
- [ ] 首次打开 App 自动弹出定位权限请求
- [ ] 点击“允许”后页面显示“已获取当前位置”
- [ ] 点击“拒绝”后页面显示“定位权限未授予”且 App 不崩溃
- [ ] 可在“总览 / 主题 / 通知配置 / 联系人 / SOS / 历史 / 工具”间正常切换
- [ ] 窄屏下导航不遮挡主要操作，页面可正常滚动

## B. 引导配置
- [ ] 电话号码可留空并可保存
- [ ] 短信号码可留空并可保存
- [ ] 短信模板可编辑并可保存
- [ ] 模板变量提示显示：`{userId} {deviceId} {lat} {lng} {time}`
- [ ] 短信预览区域会随模板内容实时变化
- [ ] 配置页可显示持久化的 `userId / deviceId`
- [ ] 可进入“联系人”页新增联系人并在列表中显示
- [ ] 可编辑联系人并保存新号码
- [ ] 可删除联系人
- [ ] 可一键将联系人号码填入电话 / 短信字段，并跳回“通知配置”页继续保存

## C. 主题与外观
- [ ] 在 Android 12+ 设备上，首次打开默认启用壁纸吸色主题
- [ ] 在“主题”页切换预设调色板后界面主色会立即变化
- [ ] 在“主题”页修改自定义颜色后界面主色会立即变化
- [ ] 重新打开 App 后主题选择仍会保留

## D. SOS 倒计时流程
- [ ] 点击“触发 SOS”进入 5 秒倒计时
- [ ] 倒计时中点击“取消 SOS”可中断发送
- [ ] 倒计时结束后自动触发 SOS

## E. SOS 触发结果
- [ ] 后端收到 SOS（日志/接口返回成功）
- [ ] 电话号码不为空时可拉起拨号界面
- [ ] 短信号码不为空时可拉起短信界面并带入模板内容
- [ ] 号码留空时相应通道显示 skipped
- [ ] 触发完成后可在“历史”页看到刚触发的事件
- [ ] 点击历史记录后可查看时间、位置、通知结果详情

## F. 构建与安装
- [ ] `npm run build` 成功
- [ ] `npm run android:sync` 成功
- [ ] `npm run android:apk` 成功
- [ ] 输出文件名包含版本号，例如 `solo-youth-safety-v0.2.0-debug.apk`
- [ ] 安装生成的版本化 APK 后可正常打开运行
