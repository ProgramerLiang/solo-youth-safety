# 项目现状评估与下一阶段 Roadmap

更新时间：2026-03-26

## 一句话结论
当前项目已经完成了 **可运行、可安装、可演示的 Android MVP**。下一阶段不再在本文件内扩写完整实施方案，只保留最小路线图索引：优先补齐可信试用所需的工程基线，并将详细分析下沉到“路线图展开 + 审查总表 + 正式整改清单”3 类入口文档。

## 当前已确认的工程优先顺序（2026-03-24）
本轮已确认采用“**先补文档，再进入实现**”的推进方式，并优先处理以下 6 个工程项：
1. [x] 前后端基础自动化测试与 CI（基础版已接入）
2. [x] 远端身份 / 鉴权基线与 CORS 收口（后端最小请求头身份基线 + CORS 白名单已落地）
3. [x] P0-2 SOS 失败恢复与用户可见状态机（已完成）
4. [ ] P0-3 Android 后台限制 / 前台服务边界澄清（下一步）
5. [ ] 继续拆分大文件、整理模块边界
6. [ ] 统一版本与配置入口 / 收敛本地后端与远端后端的 API 契约一致性

说明：
- P0-2 已完成的范围是：已覆盖定位失败、远端失败、短信失败、拨号失败、部分成功；SOS 页已有结果摘要、子步骤状态、失败原因查看与补救动作；本地 / 远端 `notifications` 已统一到细粒度状态摘要层。
- 这轮完成的本质是“用户可见状态机 + 失败恢复反馈”已落地：用户现在可以直接看到 SOS 结果摘要、每个子步骤状态、失败原因与对应补救建议，而不是只看到单一成功 / 失败结论。
- 原生短信 / 拨号状态当前只表示“已尝试发送 / 已尝试拉起”，不表示送达确认或接通确认；因此 P0-2 的完成口径是“用户可见状态机与失败恢复已落地”，不是“运营商送达回执”或“通话接通确认”。
- 统一边界声明：本轮已完成的是 **SOS 失败恢复与用户可见状态机**，以及前台 / 应用存活期间的周期采样、本地补发与手动 SOS；不是自动报警、Android 长时后台保活、被系统杀死后仍持续追踪、已具备前台服务长期运行方案、正式应急平台或完整账号体系。
- P0-3 统一边界摘要：当前可承诺能力仅为 **前台 / 应用存活期间的周期采样 + 本地补发 + 手动 SOS**；当前不承诺 Android 长时后台保活、被系统杀死后仍持续追踪、前台服务长期运行方案、自动报警 / 围栏 / 异常检测 / 正式应急平台。
- 下一步聚焦 P0-3：澄清 Android 后台限制、Doze、前台服务与被杀进程后的真实边界；当前只做边界澄清与真机验证，不提前实现前台服务长期运行方案，这不等于自动报警、后台保活、正式应急平台或完整账号体系扩展。
- 这些项的优先级当前高于联系人角色、历史地图化、风险区域提示等功能增强项。
- Android 后台限制、SOS 失败恢复、立即做 / 延后 / 近期不做的优先级矩阵，不在本文件展开，统一以下沉文档为准：`docs/mvp/ALIGNMENT_PLAN_FROM_ANDROID_MVP_TO_ORIGINAL_VISION.md`
- 权威中文审查总表统一入口：`docs/mvp/ORIGINAL_PROMISE_VS_CURRENT_STATUS.md`
- 权威中文整改统一入口：`docs/mvp/整改清单与审查入口.md`

### 为什么“隐私合规 / 授权可见”仍是最高优先级持续收口项
原始需求文档已明确提出：**“加密传输与存储，遵循最小权限原则，仅授权账号可见。”**

当前仓库现状已较前一轮前进了一步：
- `backend/main.py` 已不再使用默认全开放 CORS，而是改为通过 `SAFETY_ALLOWED_ORIGINS` 配置白名单；
- 远端受保护接口已建立最小请求头身份基线；缺失 `X-Safety-User-Id`、头体 `userId/deviceId` 不一致、或 `X-Safety-Client-Mode` 非 `remote` 时会拒绝；`GET /api/v1/health` 继续保持匿名可访问；
- 联系人、轨迹、SOS 历史、紧急配置等接口已进入 MVP 级资源归属校验边界。

但这仍然**不是正式账号体系，也不是完整隐私合规闭环**：
- 远端接口仍未建立正式登录、token、session、API key 或刷新令牌机制；
- HTTPS 部署约束、敏感字段存储策略、自动化安全回归测试仍需继续补齐；
- 因而“仅授权账号可见”“最小权限原则”目前只能表述为**已具备 MVP 级最小授权边界**，不能上升为正式上线级安全承诺。

这依然是决定项目能否从“可演示”走向“可信试用”的前置条件：
- 当前已可以明确说明远端敏感接口不是完全裸露状态；
- 但在完整账号、安全治理、回归测试未补齐前，仍不应把远端形态视为正式上线安全后端；
- 对外沟通必须持续使用“Android MVP / 联调基线 + 最小身份边界已落地，但非完整账号体系”的口径。
