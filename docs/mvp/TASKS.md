# MVP 开发任务清单

更新时间：2026-03-26

## 本轮优先任务（已确认）
本轮先做文档同步，再进入实现；当前已完成第 1、2 项，下一步进入第 3 项 **P0-3 Android 后台限制 / 前台服务边界澄清**。

## P0 / P1 / P2 实施待办总表（按当前执行优先级）

> 排序原则：优先遵循 README 与当前差异审查结论，先补 **鉴权/CORS、安全基线、SOS 失败恢复、后台限制澄清、地图化回放**，再决定是否进入围栏 / 异常检测 / 导航 / 偷拍等更高复杂度方向。
>
> 后端整改拆分规则：**P0 先收口安全边界与授权可见（鉴权 / CORS / 资源归属校验）**；**规则引擎、围栏、异常检测、分级响应、多协议上行** 统一放入后续能力扩展，不再混写成单个“大后端技术债”。

### P0（立即进入开发，阻断“可信试用”）

#### P0-1. 远端身份 / 鉴权占位与 CORS 收口
- [x] 定义 MVP 级身份方案：已明确 `userId`、`deviceId`、请求头身份声明三者关系
- [x] 后端增加最小归属校验：已实现头/体不一致拒绝、跨用户资源拒绝、缺失身份拒绝
- [x] CORS 改为按环境白名单配置，已移除远端部署默认 `*`
- [x] 前端统一注入身份请求头，并补 401/403/422 的用户可见提示
- [x] 为越权读写、头/体不一致、缺失身份、非白名单来源补接口测试
- [x] README / API 文档同步“远端模式尚未完成正式账号体系，但已具备最小授权边界”的约束口径
- [x] 明确 HTTPS 部署约束、敏感字段存储策略、上线前不可跳过项

#### P0-2. SOS 失败恢复与用户可见状态机
- [x] 已覆盖定位失败、远端失败、短信失败、拨号失败、部分成功等用户可见状态，并区分成功 / 部分成功 / 失败 / 子步骤失败原因
- [x] SOS 页已提供结果摘要、子步骤状态、失败原因查看与补救动作，避免静默失败
- [x] 已明确失败后的可执行补救方向：检查权限、刷新定位、检查网络、确认联系人号码、稍后重试；状态摘要层已能承接本地与远端结果
- [x] 已统一本地 / 远端 `notifications` 到细粒度状态摘要层，历史与本地快照持久化不会丢失 `failed / triggered / dispatched / attempted` 等结果
- [x] 已通过前端状态机构造与测试覆盖主要失败路径，避免“轨迹在补发、SOS 无感知”的静默落差
- [x] 已明确原生短信 / 拨号状态仅表示“已尝试发送 / 已尝试拉起”，不表示送达确认或接通确认

#### P0-3. Android 后台限制 / 前台服务边界澄清
> 统一边界声明：本轮已完成的是 **SOS 失败恢复与用户可见状态机**，用于把定位失败、远端失败、短信失败、拨号失败、部分成功等结果做成用户可见反馈；**不等于** 自动报警、长期后台保活、正式应急平台或完整账号体系。下一步进入 **P0-3 Android 后台限制 / 前台服务边界澄清**，继续说明 Android 真实系统限制与可承诺范围。
- [ ] 产出 Android 10+/13+/14+ 下定位、Doze、前台服务、被杀进程后的能力边界说明
- [ ] 在前端文案中明确：当前守护能力是周期采样 + 本地补发，不等于长期后台保活承诺
- [ ] 决定是否进入前台服务 PoC；若进入，仅定义短时场景与验收边界
- [ ] 形成真机验证记录，并同步 README / 测试清单 / 路线图口径
- [ ] 持续强调本阶段不是自动报警、后台保活、正式应急平台或完整账号体系扩展
